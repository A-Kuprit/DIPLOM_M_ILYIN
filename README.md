# Дипломный проект профессии «1C-программист: с нуля до middle»
 
## Доработка конфигурации «Управление IT-фирмой для компании «Ваш компьютерный мастер»

### Инструкция по начальной настройке доработанной конфигурации.

В ходе доработки конфигурации «Управление IT-фирмой для компании «Ваш компьютерный мастер» разработан и подготовлен к эксплуотации новый функциональный блок, в котором реализованы:

- новые подсистемы «Работа с заявками клиентов» и «Расчёт управленческой зарплаты» в соответствии с требованиями заказчика,
- отчёты по новой функциональности,
- автотесты для проверки корректности работы подсистемы «Работа с заявками клиентов».

Для обозначения добавленных объектов, реквизитов, методов использован префикс ВКМ_ и они размещены в новой служебной подсистеме ВКМ_ДобавленныеОбъекты.

#### Шаг 1
Создайте группу и бота в Telegram в соответствии с [описанием реализации](https://github.com/A-Kuprit/DIPLOM_M_ILYIN/tree/main/tasks/telegram.md). Они необходимы для отправки уведомлений специалистам о созданных документах «Обслуживание клиента». Уведомления создаются при создании нового документа, а также при изменениях даты и ФИО специалиста.

#### Шаг 2
В подситсеме «Работа с заявками клиентов» в разделе «Сервис» заполните следующие константы: 
- «Токен управления теленрам-ботом» - заполняется токеном бота Telegram; 
- «Идентификатор группы для оповещения» - заполняется идентификатором группы Telegram;
- «Номенклатура абонентская плата» - заполняется номенклатурой «Абонентская плата» созданной в справочнике «Номенклатура» где «Вид номенклатуры» установлен «Услуга»;
- «Номенклатура работы специалиста» - заполняется номенклатурой «Работы специалиста» созданной в справочнике «Номенклатура» где «Вид номенклатуры» установлен «Услуга».

#### Шаг 3 
Подготовьте справочники «Организации», «Контрагенты», «Номенклатура» и «Договоры контрагентов». Они необходимы при заполнения документов «Реализация товваров и услуг», «Обслуживание клиента» в подситсеме «Работа с заявками клиентов». Обязательно создайте договор с видом «Абонентское обслуживание».

#### Шаг 4
В справочнике «Пользователи» создайте для каждого сотрудника пользователя. 
Например : бухгалтер Паршина Елена, как пользователь «Паршина».
На вкладке права доступа назначьте каждому пользователю соответствующий должности 
профиль группы доступа. Наименования профиля группы доступа созданы для вашего 
удобства интуитивно понятными. Пример: для бухгалтеров профиль группы доступа – 
«Бухгалтер ИТ Фирмы», для кадровика – «Кадровик расчетчик».

#### Шаг 5
В подситсеме «Расчет управленческой зарплаты» заполните регистр сведений «Условия оплаты сотрудников», 
укажите процент от продаж у специалистов и оклады у всех сотрудников.

#### Шаг 6
В справочник «Графики работы» заведите необходимый вид графика работы, например пятидневка. 
Затем в разделе «Сервис» воспользуйтесь обработкой «Заполнение графика». 
В поле «График работы» выберите ранее созданный элемент справочника «пятидневка»,
в поле «Выбор периода» выберите период (год) для заполнения графика,
в поле «Выходные дни» через запятую перечислите порядковый номер выходного дня. 
Например: 6,7 (как суббота и воскресенье). Нажмите кнопку «Заполнить». 
Далее в регистре «График работы» проверьте корректность заполнения графика.

#### Шаг 7
Проинструктируйте бухгалтера о возможности автоматического формирования документов по реализации за месяц с помощью обработки «Массовое создание документов», находящейся в разделе «Сервис» , а так же возможности автоматического заполнения документа «Выплата зарплаты» с помощью кнопки «Заполнить» на форме документа.
